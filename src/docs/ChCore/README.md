# 为什么要写这个文档

在做 `ChCore` 的实验的时候，我觉得实验文档不够详尽，表现在：
1. 想要你完成的事情描绘的十分简介，仅仅是告诉你 “我们现在需要补充这个函数”，但这个函数的流程，这个函数的目的，甚至于很多信息都得你从这个庞大的项目中去寻找。
2. 对信息的描绘缺失很多，表现在对某个重要抽象的解释晦涩，甚至很多不解释，期望你自己去理解。

可能这样的实验方式很适合经受过良好训练的上交学子，但对于第一次接触 `OS` 课程甚至第一次接触这种难度较大的实验的学生，我认为文档的指引十分重要（例如 `MIT` 的文档会给很多 `HINT`），这种指引避免了学生盲人摸象一样的实验体验，能够有效利用学生的时间，而不是把时间花费在阅读一份完整项目的源码上（这对于第一次接触 `OS` 源码的学生是十分不友好的）

因此，我对这份实验文档进行扩写和补充，尽可能的给出不影响实验体验的提示（指不给出具体的做法），但又能确保大家能够快速理解想要你做的事情。
